Reflection Hub with Kedro
Demo recap
What you just saw
A self-improving AI agent system — from quality signal to approved improvement, fully auditable
The loop in action
Campaign
B2B Sales agent generated personalised outreach emails for each customer case
Evaluation
Scored across 7 dimensions — 4 deterministic + 3 LLM-judged, including personalisation and groundedness
Scouts
Flagged a personalisation gap and missing call-to-action — no LLM calls, pure deterministic logic
Reflection
Meta-agent proposed a targeted prompt update backed by failing cases and run history
Approve & Apply
Human approved — improved prompt committed, audit trail updated, next cycle queued
Key takeaways
Agents that get better over time
Each cycle builds on the last. Failures become evidence. New regression tests ensure the same failure cannot recur unnoticed in future runs.
↻ Reproducible improvement loop
Fast, deterministic pattern detection
Scouts identify quality regressions without a single LLM call. Signals are stored in a cross-agent index so one team's finding benefits all three agents.
⚡ Cross-agent intelligence
Human oversight without friction
Nothing changes without a human decision. Every proposal is evidence-backed, reviewable, and fully logged — transparent to engineers and executives alike.
🛡 Governance built in
What Kedro brings
Standardization
One consistent improvement framework for every use case, so evaluation patterns and approved improvements can be shared across teams.
Modularity
Separate pipelines for execution, evaluation, signal detection, reflection, and application.
Reproducibility
Traceable runs that make it possible to audit how improvements were generated and applied.
Visibility
Workflow, dependency, and data-flow views through Kedro-Viz.
Governance
Explicit approval steps, versioned changes, and reviewable I/O contracts.
